# REAP Roadmap

This roadmap is the high-level summary version of [development_phases.md](/Users/karlosiric/Documents/MyProjects/REAP/docs/development_phases.md).

## The order that matters

1. finish the engine foundation
2. bootstrap the runtime window/render path
3. build the command/config backbone
4. get a local graybox 3D loop running
5. build honest multiplayer/server-client shape
6. add the first combat loop
7. add BSP runtime
8. add VM/game-script path
9. add custom formats and tools when the runtime demands them
10. push toward a stable product

## Scope reminder

REAP is not just:
- a shooter

It is also:
- an engine runtime
- a VM
- a tools pipeline
- a data/format ecosystem

That means progress must be staged carefully.

## Practical solo rule

At any given time:
- one active milestone
- one active subsystem focus
- one or two active files

That rule matters more than ambition.
